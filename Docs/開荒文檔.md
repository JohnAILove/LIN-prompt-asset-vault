# Lin Prompt Asset Vault（小林創意提示詞與資產庫）開荒文檔

## 0. 專案名稱
- **正式英文名稱**：Lin Prompt Asset Vault
- **建議資料夾／Repo 名稱**：`lin-prompt-asset-vault`
- **簡稱**：LPAV

---

## 1. 這個專案在幹嘛
這是一個給 **小林創意** 自己使用的 **Chrome Extension（Chrome 擴充功能）** 專案。

它的目的不是做公開平台，而是做一個低摩擦、可長大的 **Prompt + Asset Vault（提示詞＋資產庫）**：
- 快速收藏提示詞
- 上傳圖片
- 上傳影片
- 把文字、圖片、影片集中到同一個索引庫
- 未來可搜尋、可回看、可複用

### 核心定位
> 一個常駐於 Chrome 的個人素材收集器，透過 Google API（應用程式介面）直接把提示詞、圖片、影片送進自己的資產庫。

---

## 2. 這件事值不值得做
**值得做，而且很符合目前階段。**

原因：
1. 你現在最缺的不是技術研究，而是可反覆利用的資產系統。
2. 你需要的是「看到／想到好東西就先收進庫」，不是每次從零開始。
3. 這個系統未來可直接支援：
   - 作品集複現
   - 提案素材回查
   - 樣片與案例管理
   - 外包交接與內部標準化

---

## 3. 專案目標
### 第一階段目標
做出一個能真的用的最小閉環：
- 可快速輸入提示詞
- 可上傳圖片
- 可上傳影片
- 可寫入 Google Sheet（Google 試算表）索引
- 可保存 Google Drive（Google 雲端硬碟）檔案連結

### 第二階段目標
- 可查看最近資料
- 可依類型／標籤搜尋
- 可點開圖片／影片回看

### 第三階段目標
- 可加入常用模板
- 可加入收藏星號
- 可加入快速複製／快速再利用

## 4. MVP（最小可行產品）定義
### 先做的功能
1. **文字收藏**
   - 輸入 prompt
   - 選類型
   - 加標籤
   - 加備註
   - 寫進 Google Sheet

2. **圖片上傳**
   - 選圖片檔
   - 上傳到 Google Drive
   - 回存 fileId / 連結到 Google Sheet

3. **影片上傳**
   - 選影片檔
   - 上傳到 Google Drive
   - 回存 fileId / 連結到 Google Sheet

4. **常駐入口**
   - Chrome 工具列圖示
   - 快捷鍵叫出 Popup（彈出視窗）

### 暫時不要做的功能
- 自動抓當前頁面網址
- 自動抓頁面標題
- YouTube 上傳
- 網頁右側浮動按鈕
- 自動分類
- 多人共用
- 漂亮後台

## 5. 技術選型
### 前端
- **Chrome Extension（Chrome 擴充功能）**
- **Manifest V3（第三版擴充功能規格）**
- Popup UI（彈出介面）

### Google 端
- **Google Drive API**：存圖片與影片
- **Google Sheets API**：存索引與欄位資料
- **chrome.identity**：取得 Google OAuth（授權）權杖

### 原則
- 不走 Google Apps Script（Google 應用程式腳本）
- 走 Google OAuth + API 直連
- 第一版先跑通，不追求完整平台化

## 6. 系統架構
```text
Chrome Extension
  ├─ Popup UI
  ├─ File Picker（檔案選擇）
  ├─ Form State（表單狀態）
  └─ Google OAuth Token

        ↓

Google APIs
  ├─ Drive API
  │   ├─ 圖片上傳
  │   ├─ 影片上傳
  │   └─ 設定公開或可讀取連結
  └─ Sheets API
      └─ 寫入 metadata（中繼資料）

        ↓

Google Drive
  ├─ /LPAV/images
  └─ /LPAV/videos

Google Sheet
  └─ assets
```

## 7. 建議 Google Sheet 結構
### 檔名
**小林創意｜提示詞與資產庫**

### 主分頁
`assets`

### 建議欄位
| 欄位 | 說明 |
|---|---|
| id | 唯一編號 |
| created_at | 建立時間 |
| asset_type | `text` / `image` / `video` |
| title | 資料標題 |
| prompt_text | 主提示詞 |
| tags | 標籤，逗號分隔 |
| notes | 備註 |
| drive_file_id | Drive 檔案 ID |
| drive_web_view_link | Drive 檢視連結 |
| mime_type | 檔案型別 |
| is_public | 是否公開 |
| favorite | 是否收藏 |
| status | `active` / `archived` |

### 後續可加分頁
- `tags`
- `settings`
- `logs`

## 8. 建議 Google Drive 結構
### 根資料夾
`LPAV`

### 子資料夾
- `images`
- `videos`

## 9. Popup（彈出視窗）畫面規格
### 主要區塊
1. Header

2. Asset Type Selector（資產類型切換）
   - Text
   - Image

3. 表單區
   - Title

   - Reset
   - View Recent

### 體驗原則
- 優先單手快速操作
- 欄位少、動作短
- 一次只做一件事

## 10. API 工作流
### A. 文字資料
1. 開啟 Popup
2. 填入 title / prompt / tags / notes
3. 呼叫 Sheets API append
4. 成功提示

### B. 圖片資料
1. 開啟 Popup
2. 選圖檔
3. 呼叫 Drive API upload
4. 取得 `fileId`
5. 依設定決定是否設公開權限
6. 呼叫 Sheets API append 寫入索引
7. 成功提示

### C. 影片資料
1. 開啟 Popup
2. 選影片檔
3. 呼叫 Drive API upload
4. 影片較大時改用 resumable upload（可續傳上傳）
5. 取得 `fileId`
6. 依設定決定是否設公開權限
7. 呼叫 Sheets API append 寫入索引
8. 成功提示

## 14. 風險與注意事項
1. **先不要做太多欄位**
   欄位一多，使用意願會下降。

2. **先不要碰 YouTube**
   它是發布流程，不是目前最省事的資產庫流程。

3. **影片先求能上傳，不先求超快播放**
   第一版是資產收納，不是影音平台。

4. **先把流程跑通，再談 UI 升級**
   後面再考慮右側浮動按鈕、側邊面板。
